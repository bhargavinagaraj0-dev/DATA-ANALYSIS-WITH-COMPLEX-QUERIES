# DESCRIPTION:-

The objective of this task is to perform advanced data analysis using complex SQL queries. In real-world databases, simple queries are often not sufficient to extract meaningful insights. Therefore, advanced SQL concepts such as window functions, subqueries, and Common Table Expressions (CTEs) are used to analyze data more effectively and identify trends or patterns.
In this task, different SQL techniques were applied to work with structured data stored in relational tables. The focus was on analyzing the data rather than just retrieving it.

The following concepts were used:

1)Window Functions – These functions are used to perform calculations across a set of rows related to the current row without grouping the result into a single output row. Examples include functions like ROW_NUMBER(), RANK(), and SUM() OVER(). These are useful for tasks such as ranking employees, calculating running totals, and comparing values within partitions.

2)Subqueries – Subqueries are queries written inside another query. They are used to break down complex problems into smaller parts and make queries more flexible. Subqueries can be used in SELECT, WHERE, and FROM clauses to filter or transform data based on intermediate results.

3)Common Table Expressions (CTEs) – CTEs are temporary result sets defined using the WITH clause. They improve readability and structure of complex queries by breaking them into logical steps. CTEs are especially useful for performing multi-step data transformations and recursive queries.
